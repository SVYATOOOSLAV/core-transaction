# Transactions API

Базовый путь: `/api/v1/transactions`

## Общие принципы

- **Идемпотентность**: все операции создания транзакций принимают `idempotencyKey` (UUID). Если транзакция с таким ключом уже существует, возвращается ранее созданный результат без повторного выполнения.
- **Пессимистичная блокировка**: при списании средств счет-источник блокируется через `PESSIMISTIC_WRITE` для предотвращения гонок.
- **Транзакционность**: все операции выполняются внутри `@Transactional` — при ошибке изменения откатываются.
- **Точность**: суммы хранятся как `BigDecimal` с точностью `NUMERIC(19,4)`.

## Общий формат ответа (TransactionResponse)

| Поле                 | Тип              | Описание                              |
|----------------------|------------------|---------------------------------------|
| id                   | Long             | ID транзакции                         |
| idempotencyKey       | UUID             | Ключ идемпотентности                  |
| transactionType      | String           | Тип транзакции (enum name)            |
| status               | String           | Статус: PENDING, COMPLETED, FAILED, CANCELLED |
| sourceAccountId      | Long?            | ID счета-источника (null для кредитных операций) |
| destinationAccountId | Long?            | ID счета-назначения                   |
| amount               | BigDecimal       | Сумма                                 |
| currency             | String           | Валюта                                |
| description          | String?          | Описание                              |
| errorMessage         | String?          | Сообщение об ошибке (при FAILED)      |
| createdAt            | OffsetDateTime   | Дата создания                         |
| completedAt          | OffsetDateTime?  | Дата завершения                       |

## POST `/api/v1/transactions/savings` — Перевод на накопительный счет

### Описание

Внутренний перевод средств с любого счета на счет типа SAVINGS.

### Входные параметры (Request Body — TransferRequest)

| Поле                 | Тип        | Обязательное | Валидация       | Описание                     |
|----------------------|------------|:------------:|-----------------|------------------------------|
| idempotencyKey       | UUID       | да           | NotNull         | Ключ идемпотентности         |
| sourceAccountId      | Long       | да           | NotNull         | ID счета-источника           |
| destinationAccountId | Long       | да           | NotNull         | ID счета-назначения (SAVINGS)|
| amount               | BigDecimal | да           | NotNull, >= 0.01 | Сумма перевода              |
| description          | String?    | нет          | —               | Описание перевода            |

### Логика работы

1. Проверка идемпотентности по `idempotencyKey`.
2. Проверка существования и активности счета-назначения.
3. Проверка, что тип счета-назначения = `SAVINGS`. Иначе — ошибка.
4. Пессимистичная блокировка счета-источника (`findByIdForUpdate`).
5. Проверка активности счета-источника и достаточности баланса.
6. Списание суммы со счета-источника.
7. Зачисление суммы на счет-назначения.
8. Создание транзакции со статусом `COMPLETED` и типом `TRANSFER_SAVINGS`.

### Ошибки

| HTTP-код | Описание                                |
|----------|-----------------------------------------|
| 400      | Ошибка валидации / недостаточно средств |
| 404      | Счет не найден                          |
| 422      | Тип счета-назначения не SAVINGS         |

## POST `/api/v1/transactions/interbank` — Межбанковский перевод по номеру карты

### Описание

Перевод средств между картами, в том числе в другие банки. Поиск счетов осуществляется по номерам карт.

### Входные параметры (Request Body — InterbankTransferRequest)

| Поле                   | Тип        | Обязательное | Валидация       | Описание                       |
|------------------------|------------|:------------:|-----------------|--------------------------------|
| idempotencyKey         | UUID       | да           | NotNull         | Ключ идемпотентности           |
| sourceCardNumber       | String     | да           | NotBlank        | Номер карты отправителя        |
| destinationCardNumber  | String     | да           | NotBlank        | Номер карты получателя         |
| amount                 | BigDecimal | да           | NotNull, >= 0.01 | Сумма перевода                |
| description            | String?    | нет          | —               | Описание перевода              |

### Логика работы

1. Проверка идемпотентности.
2. Поиск карты отправителя по `sourceCardNumber`. Если не найдена — ошибка.
3. Поиск карты получателя по `destinationCardNumber`. Если не найдена — ошибка.
4. Получение счетов, привязанных к картам.
5. Пессимистичная блокировка счета-источника.
6. Проверка активности и достаточности баланса.
7. Списание и зачисление.
8. Создание транзакции с типом `INTERBANK_TRANSFER` и статусом `COMPLETED`.

### Ошибки

| HTTP-код | Описание                                |
|----------|-----------------------------------------|
| 400      | Ошибка валидации / недостаточно средств |
| 404      | Карта или счет не найдены               |

---

## POST `/api/v1/transactions/sbp` — Перевод через СБП

### Описание

Перевод по номеру телефона получателя через Систему быстрых платежей. Средства зачисляются на расчетный (CHECKING) счет получателя.

### Входные параметры (Request Body — SbpTransferRequest)

| Поле                  | Тип        | Обязательное | Валидация       | Описание                     |
|-----------------------|------------|:------------:|-----------------|------------------------------|
| idempotencyKey        | UUID       | да           | NotNull         | Ключ идемпотентности         |
| sourceAccountId       | Long       | да           | NotNull         | ID счета-отправителя         |
| recipientPhoneNumber  | String     | да           | NotBlank        | Номер телефона получателя    |
| amount                | BigDecimal | да           | NotNull, >= 0.01 | Сумма перевода              |
| description           | String?    | нет          | —               | Описание перевода            |

### Логика работы

1. Проверка идемпотентности.
2. Поиск пользователя-получателя по номеру телефона. Если не найден — ошибка.
3. Поиск расчетного счета (CHECKING) получателя. Если не найден — ошибка.
4. Пессимистичная блокировка счета-источника.
5. Проверка активности и достаточности баланса.
6. Списание со счета-источника, зачисление на CHECKING счет получателя.
7. Создание транзакции с типом `SBP_TRANSFER` и статусом `COMPLETED`.

### Ошибки

| HTTP-код | Описание                                      |
|----------|-----------------------------------------------|
| 400      | Ошибка валидации / недостаточно средств       |
| 404      | Получатель или его расчетный счет не найдены   |

## POST `/api/v1/transactions/gift` — Денежный подарок

### Описание

Кредитная операция — зачисление средств на счет без списания с какого-либо источника. Используется для денежных подарков.

### Входные параметры (Request Body — MoneyGiftRequest)

| Поле                 | Тип        | Обязательное | Валидация       | Описание                  |
|----------------------|------------|:------------:|-----------------|---------------------------|
| idempotencyKey       | UUID       | да           | NotNull         | Ключ идемпотентности      |
| destinationAccountId | Long       | да           | NotNull         | ID счета-получателя       |
| amount               | BigDecimal | да           | NotNull, >= 0.01 | Сумма зачисления         |
| description          | String?    | нет          | —               | Описание                  |

### Логика работы

1. Проверка идемпотентности.
2. Проверка существования и активности счета-назначения.
3. Зачисление суммы на счет (без списания с источника).
4. Создание транзакции с типом `MONEY_GIFT`, `sourceAccount = null`, статус `COMPLETED`.

### Ошибки

| HTTP-код | Описание                     |
|----------|------------------------------|
| 400      | Ошибка валидации             |
| 404      | Счет-назначения не найден    |

## GET `/api/v1/transactions/account/{accountId}` — Получение транзакций по счету

### Описание

Возвращает все транзакции, где указанный счет является источником или назначением.

### Входные параметры

| Параметр  | Тип  | Источник     | Описание  |
|-----------|------|-------------|-----------|
| accountId | Long | Path Variable | ID счета |

### Выходные параметры

`List<TransactionResponse>` — массив транзакций (может быть пустым).

### Логика работы

1. Поиск всех транзакций, где `sourceAccountId = accountId` ИЛИ `destinationAccountId = accountId`.
2. Маппинг каждой сущности в DTO.
3. Возврат списка.

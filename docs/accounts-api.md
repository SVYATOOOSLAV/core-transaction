# Accounts API

Базовый путь: `/api/v1/accounts`

---

## POST `/api/v1/accounts` — Создание счета

### Описание

Создает новый банковский счет для существующего пользователя. Баланс инициализируется нулем, счет активен по умолчанию.

### Входные параметры (Request Body)

| Поле          | Тип    | Обязательное | Валидация         | Описание                              |
|---------------|--------|:------------:|-------------------|---------------------------------------|
| userId        | Long   | да           | NotNull           | ID владельца счета                    |
| accountNumber | String | да           | NotBlank, max 20  | Номер счета (уникальный)              |
| accountType   | String | да           | NotBlank          | Тип счета: CHECKING, SAVINGS, DEPOSIT, BROKERAGE |
| currency      | String | нет          | NotBlank, max 3   | Валюта (по умолчанию `"RUB"`)         |

### Пример запроса

```json
{
  "userId": 1,
  "accountNumber": "40817810000000001",
  "accountType": "CHECKING",
  "currency": "RUB"
}
```

### Выходные параметры (Response Body)

| Поле          | Тип            | Описание              |
|---------------|----------------|-----------------------|
| id            | Long           | ID счета              |
| userId        | Long           | ID владельца          |
| accountNumber | String         | Номер счета           |
| accountType   | String         | Тип счета             |
| currency      | String         | Валюта                |
| balance       | BigDecimal     | Баланс (0.0000)       |
| isActive      | Boolean        | Активен ли счет       |
| createdAt     | OffsetDateTime | Дата создания         |

### Пример ответа

```json
{
  "id": 1,
  "userId": 1,
  "accountNumber": "40817810000000001",
  "accountType": "CHECKING",
  "currency": "RUB",
  "balance": 0.0000,
  "isActive": true,
  "createdAt": "2026-03-18T12:00:00+03:00"
}
```

### Логика работы

1. Валидация полей запроса.
2. Проверка существования пользователя по `userId`. Если не найден — **404 Not Found**.
3. Проверка уникальности номера счета. Если дублируется — ошибка (unique constraint).
4. Создание сущности `AccountEntity` с балансом `BigDecimal.ZERO` и `isActive = true`.
5. Сохранение в БД, маппинг в DTO и возврат.

### Ошибки

| HTTP-код | Описание                          |
|----------|-----------------------------------|
| 400      | Ошибка валидации входных данных   |
| 404      | Пользователь не найден            |
| 409      | Счет с таким номером уже существует |

## GET `/api/v1/accounts/{id}` — Получение счета по ID

### Входные параметры

| Параметр | Тип  | Источник     | Описание  |
|----------|------|-------------|-----------|
| id       | Long | Path Variable | ID счета |

### Выходные параметры

`AccountResponse` — аналогично ответу создания.

### Логика работы

1. Поиск счета по ID.
2. Если не найден — **404 Not Found**.
3. Маппинг в DTO и возврат.

### Ошибки

| HTTP-код | Описание        |
|----------|-----------------|
| 404      | Счет не найден  |

---

## GET `/api/v1/accounts/user/{userId}` — Получение всех счетов пользователя

### Входные параметры

| Параметр | Тип  | Источник     | Описание         |
|----------|------|-------------|------------------|
| userId   | Long | Path Variable | ID пользователя |

### Выходные параметры

`List<AccountResponse>` — массив счетов пользователя.

### Логика работы

1. Поиск всех счетов по `userId`.
2. Маппинг каждой сущности в DTO.
3. Возврат списка (может быть пустым).

# Users API

Базовый путь: `/api/v1/users`

---

## POST `/api/v1/users` — Создание пользователя

### Описание

Создает нового пользователя в системе. Номер телефона должен быть уникальным.

### Входные параметры (Request Body)

| Поле          | Тип      | Обязательное | Валидация        | Описание              |
|---------------|----------|:------------:|------------------|-----------------------|
| firstName     | String   | да           | NotBlank, max 100 | Имя                  |
| lastName      | String   | да           | NotBlank, max 100 | Фамилия              |
| patronymic    | String?  | нет          | max 100          | Отчество              |
| phoneNumber   | String   | да           | NotBlank, max 20  | Номер телефона (уникальный) |
| email         | String?  | нет          | Email, max 150   | Электронная почта     |

### Пример запроса

```json
{
  "firstName": "Иван",
  "lastName": "Петров",
  "patronymic": "Сергеевич",
  "phoneNumber": "+79001234567",
  "email": "ivan@example.com"
}
```

### Выходные параметры (Response Body)

| Поле        | Тип              | Описание              |
|-------------|------------------|-----------------------|
| id          | Long             | ID пользователя       |
| firstName   | String           | Имя                   |
| lastName    | String           | Фамилия               |
| patronymic  | String?          | Отчество              |
| phoneNumber | String           | Номер телефона        |
| email       | String?          | Электронная почта     |
| createdAt   | OffsetDateTime   | Дата создания         |

### Пример ответа

```json
{
  "id": 1,
  "firstName": "Иван",
  "lastName": "Петров",
  "patronymic": "Сергеевич",
  "phoneNumber": "+79001234567",
  "email": "ivan@example.com",
  "createdAt": "2026-03-18T12:00:00+03:00"
}
```

### Логика работы

1. Валидация полей запроса (Jakarta Bean Validation).
2. Проверка уникальности номера телефона. Если номер уже существует — возвращается **409 Conflict**.
3. Сохранение пользователя в БД.
4. Маппинг сущности в DTO и возврат ответа.

### Ошибки

| HTTP-код | Описание                                     |
|----------|----------------------------------------------|
| 400      | Ошибка валидации входных данных              |
| 409      | Пользователь с таким номером телефона уже существует |

## GET `/api/v1/users/{id}` — Получение пользователя по ID

### Входные параметры

| Параметр | Тип  | Источник     | Описание         |
|----------|------|-------------|------------------|
| id       | Long | Path Variable | ID пользователя |

### Выходные параметры

`UserResponse` — аналогично ответу создания.

### Логика работы

1. Поиск пользователя по ID в БД.
2. Если не найден — **404 Not Found**.
3. Маппинг в DTO и возврат.

### Ошибки

| HTTP-код | Описание               |
|----------|------------------------|
| 404      | Пользователь не найден |
